# DevPilot 产品实现计划

这份文档基于我们前面的讨论，统一了以下结论：

- `DevPilot` 是一个产品，不是“标注工具”和 “MCP 产品”两套系统
- 产品应分为两种模式：
  - `本地模式`：标注、复制、导出给 Claude / Codex / Cursor
  - `连接模式`：同步、源码命中、修复请求、状态回写
- `会话面板` 不应成为当前主流程的一级入口，尤其在本地模式下
- `稳定性副驾` 是可选增强功能，由设置面板中的单一开关启用
- `稳定性副驾` 一旦启用，就自动开启观察，不再单独暴露“自动观察”开关
- 产品主线应始终围绕：

`页面问题 -> 上下文 -> AI 任务 -> 修复 -> 回写`

---

## 一、最终产品定义

### 产品一句话

`DevPilot` 是一个把浏览器问题转成可执行前端工程任务的副驾。

### 产品分层

#### 1. 本地模式

目标：

- 在不接任何后端的情况下立即可用
- 支持页面标注和问题导出
- 作为“问题采集器”独立成立

能力：

- 元素 / 文本 / 区域标注
- 页面内 marker 可视化
- 轻量详情查看与编辑
- 一键复制给 AI
- 可选开启稳定性副驾

#### 2. 连接模式

目标：

- 把问题从“记录”推进到“处理”
- 和 MCP / AI 修复工作流打通

能力：

- Session 同步
- SSE 状态同步
- 源码候选命中
- 修复请求
- 状态回写
- 后续统一时间线

---

## 二、产品体验原则

### 原则 1：标注是入口，不是产品本体

用户不是为了“多画几个框”而来，而是为了把问题说清楚并推进解决。

### 原则 2：默认流程必须简单

不接 MCP 时，用户也应该能顺畅完成：

`标注 -> 复制 -> 去 AI`

### 原则 3：增强能力按需出现

稳定性、副驾、连接状态、修复流，不应在所有场景下都压到主界面里。

### 原则 4：少讲系统概念，多讲用户任务

少用：

- Session
- MCP
- SSE
- Schema

多用：

- 记录问题
- 复制给 AI
- 发送修复请求
- 查看处理状态

---

## 三、信息架构调整方案

## 1. 工具条

### 当前问题

- 一级入口偏多
- “会话”对本地模式用户价值不高
- 稳定性副驾不一定是所有用户都需要

### 调整后

#### 默认本地模式

工具条显示：

- `标注`
- `诊断`
- `复制给 AI`

说明：

- 不显示 `会话`
- 默认不显示 `稳定性`
- 复制能力直接成为主入口

#### 开启稳定性副驾后

工具条显示：

- `标注`
- `稳定性`
- `诊断`
- `复制给 AI`

#### 连接模式下

工具条显示：

- `标注`
- `稳定性`（如果启用）
- `处理流` 或 `进度`
- `诊断`

说明：

- 这里不建议继续用“会话”作为产品文案
- 更适合用 `处理流`、`进度`、`修复流` 之类任务导向名称

---

## 2. 标注体验

### 当前问题

- 用户标注完成后，不一定知道下一步做什么
- 会话列表和页面 marker 存在信息重复

### 调整后

#### 页面上

- 继续保留 marker 作为空间锚点
- marker 承担“回看标注”的主要入口

#### 详情动作

本地模式下，标注详情只保留核心动作：

- 编辑
- 删除
- `复制给 AI`

连接模式下：

- 编辑
- 删除
- `发送修复请求`

### 不做的事

- 不把本地模式用户强行带进“会话列表”
- 不要求用户先理解 annotation list 再开始工作

---

## 3. 稳定性副驾

### 结论

稳定性功能通过设置面板中的单一开关启用：

- `启用稳定性副驾`

### 规则

开启后：

- 工具条显示 `稳定性`
- 自动开始观察
- 开始捕获 JS / Promise / fetch / xhr 问题
- 稳定性问题进入问题列表和后续修复流

关闭后：

- 工具条隐藏 `稳定性`
- 停止观察器
- 不再采集稳定性问题

### 不再保留

- 单独的 `自动观察` 开关

### 关键反馈

用户开启稳定性副驾后，界面需要立即反馈：

- `已开始观察页面运行时问题`
- 工具条 `稳定性` 入口出现
- 新问题出现时给出 badge 或轻提示

---

## 4. 设置面板

### 角色重新定义

当前“设置面板”本质上更像：

- 功能开关面板
- 连接诊断面板

### 建议内容

#### 功能

- `启用稳定性副驾`

#### 诊断

- 当前模式：本地 / 已连接
- MCP 状态
- SSE 状态
- 当前 session
- 最近一次同步时间
- 最近一次采集时间
- 最近一次错误原因

### 文案建议

按钮名称建议从 `设置` 改为：

- `诊断`

如果仍保留“设置”名称，也建议面板内分区为：

- `功能`
- `连接诊断`

## Phase 1：收缩本地模式主流程

### 目标

让本地模式只保留最核心的使用链路：

`标注 -> 复制给 AI`

### 已完成的改动

- ✅ 工具条移除 `会话`
- ✅ 增加显著的 `复制给 AI` 主入口
- ✅ 标注详情里保留轻量动作（编辑、删除）
- ⏭️ ~~标注详情里增加 `复制给 AI`~~ — **已移除，不在当前计划内**

### 建议改动文件

- `packages/devpilot/src/view.tsx`
- `packages/devpilot/src/ui/session-panel.tsx`
- `packages/devpilot/src/hooks/use-annotations.ts`

### 验收标准

- 首次用户无需打开“会话面板”也能完成完整主流程
- 页面 marker 足以支撑标注回看和编辑

---

## Phase 2：稳定性副驾按需启用

### 目标

把稳定性能力从“默认主入口”改成“设置中启用的增强能力”。

### 要做的事

- 设置面板新增：
  - `启用稳定性副驾`
- 开启后自动注册观察器
- 工具条按开关动态显示 `稳定性`
- 抓到问题时显示 badge / 轻提示
- 移除独立“自动观察”开关

### 建议改动文件

- `packages/devpilot/src/ui/settings-panel.tsx`
- `packages/devpilot/src/ui/stability-panel.tsx`
- `packages/devpilot/src/hooks/use-stability.ts`
- `packages/devpilot/src/view.tsx`
- `packages/devpilot/src/storage.ts`

### 验收标准

- 用户只需要做一个决策：要不要启用稳定性副驾
- 开启后能立刻感知它已开始工作
- 关闭后相关入口和监听全部消失

## Phase 4：统一 AI handoff

### 目标

让“复制给 AI”和“发送修复请求”共享统一任务模型，而不是两套平行输出。

### 要做的事

- 本地模式：
  - `复制给 AI` 输出 task packet
- 连接模式：
  - `发送修复请求` 基于 task packet + MCP 命中结果
- 统一 annotation / stability / repair 的导出语义

### 建议改动文件

- `packages/devpilot/src/output.ts`
- `packages/devpilot/src/stability-output.ts`
- `packages/devpilot/src/task-packet.ts`
- `packages/devpilot/src/types/repair.ts`
- `packages/devpilot-mcp/src/**`

### 验收标准

- 不论本地还是连接模式，交给 AI 的输入结构一致
- 只是连接模式多了“状态管理和回写”

---

## Phase 5：体验打磨与信任感增强

### 目标

让用户真正相信产品“正在帮我推进问题解决”。

### 要做的事

- 自动观察开启后的即时反馈
- 最近一次采集成功/失败说明
- 连接断开 / 重连中的清晰提示
- 页面上轻量 toast / badge 反馈
- 首次使用 onboarding

### 建议改动文件

- `packages/devpilot/src/view.tsx`
- `packages/devpilot/src/ui/settings-panel.tsx`
- `packages/devpilot/src/ui/stability-panel.tsx`

### 验收标准

- 用户能快速回答：
  - 它现在有没有工作？
  - 它刚刚有没有抓到问题？
  - 我现在下一步该做什么？

## 六、成功标准

如果这份计划执行正确，最终应该达到：

- 本地模式用户不需要理解“会话 / MCP / SSE”也能顺畅使用
- 稳定性副驾只在需要时出现，不干扰主流程
- 连接模式明显比本地模式“更强”，但不是“另一个产品”
- 用户能从页面问题自然进入 AI 修复流程

---

## 七、最终判断

当前 `DevPilot` 最正确的产品形态，不是功能越多越好，而是：

- 本地模式极简好用
- 连接模式逐步增强
- 稳定性副驾按需启用
- 标注只是入口，处理闭环才是价值核心

一句话总结：

`先把”标注 -> 复制给 AI”做到极顺，再按需叠加稳定性副驾能力。`
